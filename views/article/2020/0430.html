springboot 自动装配原理分析

juno chan 2020-04-30 springboot

​ 在 SpringBoot 还没出现之前，我们开发一个 web 应用程序需要进行一系列繁琐的配置，而不是专注于业务的实现。但同时，SpringBoot 也不是什么新技术，而是基于原有的 spring 框架而开发出来的框架。

​ Spring 的 IOC 和 AOP 都是我们所熟悉的，IOC 又分为DI 和 DL ，主要用来替我们管理对象的创建、关联的依赖等。虽然很好用，但是要配置一些令人生怨的配置，所以接下来我会用几个例子来一步步的讲解 SpringBoot 是怎么为我们来自动加载所需的 bean，来简化我们的配置的。

# @SpringBootApplication

# @ComponentScan

凡是有 @Component/@Controller/@Service/@Repository/... 注解的类都会自动加载到 Spring 的容器中，默认扫描同一包下的所有类。

@ComponentScan
public class DogMain {

    public static void main(String[] args) {
        AnnotationConfigApplicationContext cx = new AnnotationConfigApplicationContext(DogMain.class);
        // 输出扫描到的类名
        String[] names = cx.getBeanDefinitionNames();
        for (String name : names) {
            System.out.println(name);
        }
    }

}

public class AutoImportSelector implements ImportSelector {

    public static final String DOG_RESOURCE_LOCATION = "META-INF/dog.factories";

    @Override
    public String[] selectImports(AnnotationMetadata annotationMetadata) {
        // 可以通过 AnnotationMetadata 获取到注解的 一些元数据
        String name = EnableDefineService.class.getName();
        Map<String, Object> metadata = annotationMetadata.getAnnotationAttributes(name);
        // 这里可以通过加载外部文件的方式来加载 Class
        List<String> autoConfigs = getDogConfigurations();
        // 这里可以做一些判断，筛选我们所需的 bean，比如排除一些 Class
        List<String> excludes = getExcludes(metadata);
        // 去掉要排除的 Class
        autoConfigs.removeAll(excludes);
        return StringUtils.toStringArray(autoConfigs);
    }
    
    private List<String> getExcludes(Map<String, Object> metadata) {
        Assert.notNull(metadata, () -> "No dog-configuration attributes found");
        AnnotationAttributes attributes = AnnotationAttributes.fromMap(metadata);
        Class<?>[] excludeClasses = attributes.getClassArray("exclude");
        return Arrays.stream(excludeClasses).map(Class::getName).collect(Collectors.toList());
    }

    private static List<String> getDogConfigurations() {
        List<String> classNameList = new ArrayList<>();
        try {
            Enumeration<URL> urls = ClassLoader.getSystemResources(DOG_RESOURCE_LOCATION);
            while (urls.hasMoreElements()) {
                URL url = urls.nextElement();
                UrlResource resource = new UrlResource(url);
                Properties properties = PropertiesLoaderUtils.loadProperties(resource);
                for (Map.Entry<?, ?> entry : properties.entrySet()) {
                    String[] factoryTypeValues = StringUtils.commaDelimitedListToStringArray((String) entry.getValue());
                    Collections.addAll(classNameList, factoryTypeValues);
                }
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        return classNameList.stream().filter(name -> !StringUtils.isEmpty(name.trim())).collect(Collectors.toList());
    }


}

# AutoConfigurationPackages.Registrar

与 ImportSelector 类似，ImportBeanDefinitionRegistrar 同样可以按条件来动态注册 bean，

相当于 xml 写法的 <bean name="" class="” />

public class LogDefinitionRegistrar implements ImportBeanDefinitionRegistrar {

    @Override
    public void registerBeanDefinitions(AnnotationMetadata importingClassMetadata, BeanDefinitionRegistry registry) {
        Class<LogService> logServiceClass = LogService.class;
        String name = StringUtils.unqualify(logServiceClass.getName());
        RootBeanDefinition logServiceBean = new RootBeanDefinition(logServiceClass);
        registry.registerBeanDefinition(name,logServiceBean);
    }
}

# AutoConfiguration 自动装配

在前面，我们通过实现 ImportSelector 来动态加载类时，是去读取一个外部文件里面的属于org.springframework.boot.autoconfigure.EnableAutoConfiguration 的类，同理 SpringBoot

也是去加载这么一个文件，通过里面的一些约定俗成的 key 来做不同的事情，这就是SpringBoot的 SPI机制，

相同做法的有 jdbc 等，它没有实现所有的数据库对应的引擎，而是也提供一个SPI来让各个数据库生产商去实现它。

# @ConditionalOnXxx

对 Spring @Conditional 的扩展，为条件注解，只有当满足某些条件时才会去加载，比如常见的 @ConditionalOnBean 表示只有加载了某个类后才会去加载本类

我们除了可以在类或方法上加这种注解，也可以在 /resources/META-INF目录下新建一个spring-autoconfigure-metadata.properties文件

com.juno.springboot.autoconfigure.demo3.CacheService.ConditionalOnBean=com.juno.springboot.autoconfigure.demo3.LogService
1

上面的例子表示只有当 LogService这个类加载了，才会去加载 CacheService
